Ironing Service 

โปรเจกต์นี้เป็นระบบจัดการร้านรีดผ้าในหมู่บ้าน ที่ออกแบบมาเพื่อแก้ปัญหาการจดบันทึกด้วยมือ โดยเปลี่ยนมาใช้ระบบ Digital ที่สามารถบันทึกข้อมูลลูกค้า ตรวจสอบการชำระเงิน และแจ้งเตือนสถานะงานผ่าน LINE OA ได้อย่างถูกต้องและแม่นยำ

## 🚀 คุณสมบัติหลักของระบบ (Key Features)
ระบบนี้ถูกออกแบบมาเพื่อรองรับกระบวนการทำงานหลักดังนี้:
- **การรับและบันทึกรายการผ้า:** รับข้อมูลลูกค้าและจำนวนผ้า พร้อมบันทึกลงฐานข้อมูลโดยอัตโนมัติ
- **การจัดการงานบริการ:** ตรวจสอบสภาพผ้า แจ้งจุดชำรุด และขออนุมัติการซ่อมแซมจากลูกค้า
- **การตรวจสอบการชำระเงิน:** ตรวจสอบยอดเงินจากสลิปโอนเงิน (Slip Verification)
- **การแจ้งเตือนและส่งมอบ:** แจ้งเตือนสถานะงานให้ลูกค้าทราบผ่าน LINE OA เมื่อผ้าพร้อมส่ง

## 🛠 เทคโนโลยีที่ใช้ (Technology Stack)
- **Google Apps Script:** สำหรับเขียน Backend ของระบบ
- **Google Sheets:** ใช้เป็นฐานข้อมูล (Database) หลักของระบบ
- **LINE Messaging API:** สำหรับการสื่อสารและแจ้งเตือนลูกค้า
- **System Analysis & Design:** ออกแบบด้วย DFD (Level 0, 1, 2) และ Use Case (BUC, SUC)

## 📂 โครงสร้างเอกสาร (Documentation)
โปรเจกต์นี้มีการออกแบบระบบอย่างเป็นขั้นตอน:
- **DFD (Data Flow Diagrams):** ครอบคลุมตั้งแตระดับ Context Diagram จนถึงระดับ Primitive Process
- **Use Cases:** ครอบคลุม Business Use Case (BUC) และ System Use Case (SUC) เพื่อแสดงขั้นตอนการทำงานของ Actor แต่ละส่วน
